# Backend Onboarding ‚Äî Configurable interactive guide

This skill guides a new developer through implementing a backend feature, end to end. It is **configurable**: refactoring proposals and reference files are parameterizable via `config.json`.

**This skill is backend-only.** It does not handle UI tests.

---

## Bootstrap ‚Äî Reading the configuration

**Step 1**: Read the configuration file:
- Path: `${CLAUDE_SKILL_DIR}/../../config.json`

Parse the JSON. This file determines:
- `proposals`: which refactoring proposals to include in phase 6 (true/false)
- `references`: where to find reference files (null = bundled, string path = project file)

**Step 2**: Reference resolution.

For each entry in `references.skills`:
- If the value is `null` ‚Üí the bundled file is at `${CLAUDE_SKILL_DIR}/../../references/skills/{key}.md`
- If the value is a string path ‚Üí read that file from the project

For each entry in `references.examples`:
- If the value is `null` ‚Üí the bundled file is at `${CLAUDE_SKILL_DIR}/../../references/examples/{key}.ts`
  - Key to bundled filename mapping:
    - `testApp` ‚Üí `testApp.ts`
    - `commandResult` ‚Üí `commandResult.ts`
    - `dsl` ‚Üí `createUserAccount.dsl.ts`
    - `e2eDriver` ‚Üí `createUserAccount.e2eDriver.ts`
    - `e2eSpec` ‚Üí `createUserAccount.e2e-spec.ts`
    - `useCase` ‚Üí `createUserAccount.useCase.ts`
    - `controller` ‚Üí `createUserAccount.controller.ts`
    - `entity` ‚Üí `userAccount.entity.ts`
    - `valueObject` ‚Üí `userAcountEmail.vo.ts`
    - `inMemoryRepository` ‚Üí `createUserAccount.inMemoryRepository.ts`
    - `useCaseDriver` ‚Üí `createUserAccount.useCaseDriver.ts`
- If the value is a string path ‚Üí read that file from the project

## Strict rules

1. **Backend-only** ‚Äî refuse tickets with `ui-test` label
2. **Always e2e** ‚Äî regardless of the label (`backend-usecase-test` or `backend-e2e-test`), generate an e2e test
3. **Use `TestApp`** ‚Äî never `PostgreSqlContainer` directly
4. **Everything in English** ‚Äî explanations, messages, comments (unless `config.language` is `"fr"`)
5. **NEVER write feature code** for the developer (Phases 4/5) ‚Äî this is educational
6. **NEVER skip the explanations** in Phase 3 ‚Äî each point must be individually confirmed
7. **NEVER proceed to Phase 6** if tests don't pass in Phase 5
8. **All refactoring proposals are optional** ‚Äî respect "skip" and "I don't want to refactor"
9. **Reference existing code** ‚Äî read and show excerpts from reference files (bundled or project), not theoretical code
10. **Guide, don't do** ‚Äî in Phase 6, explain and show examples, let the dev code
